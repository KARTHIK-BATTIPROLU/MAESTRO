## 6. Architecture Facts

Three services (per README diagram, [README.md:25-52](README.md#L25-L52)):
- **Frontend (React, :5173)** → calls **Backend** over HTTP (`axios`, base `VITE_API_URL`). Vite dev proxies `/api` → `:5000` ([vite.config.js:8-14](frontend/src/../vite.config.js#L8-L14)).
- **Backend (Express, :5000)** → API gateway. Forwards to agent service via `axios` client ([services/agentService.js:11-20](backend/src/services/agentService.js#L11-L20)); reads/writes MongoDB via Mongoose models.
- **Agent service (FastAPI, :8000)** → runs the decision pipeline (CrewAI + Gemini, and/or deterministic engine).

Connections:
- Backend → Agent: `AgentServiceClient` posts to `/process-inventory-decision`, `/start-session`, `/respond`, `/process`, `/health`, `/questions`, `/session/...` ([agentService.js](backend/src/services/agentService.js)).
- Backend `/api/inventory-decision` enriches the payload with **live business state from Mongo** before forwarding ([agentService.js:144-189](backend/src/services/agentService.js#L144-L189)).
- Agent-service internal module dependency chain: `risk_signals.py → decision_engine.py → inventory_decision_agent.py`; `orchestrator.py` composes agents (`agents.py`), tasks (`tasks.py`), prompts (`prompt_templates.py`), schemas (`schemas.py`), constants (`constants.py`) ([orchestrator.py:25-44](agent-service/orchestrator.py#L25-L44)).
- Resilience: both services have circuit breakers + retry-with-backoff ([agent-service/resilience.py](agent-service/resilience.py), [backend/src/services/resilience.js](backend/src/services/resilience.js)).

Agent pipeline (agent-service, per docstrings [main.py:5-22](agent-service/main.py#L5-L22), [orchestrator.py:5-23](agent-service/orchestrator.py#L5-L23)):
0. Raw answer preservation (q1–q10 single source of truth)
1. Router/Context-Summarization Agent **[LLM]**
2. Signal conversion (categorical → numeric) **[deterministic]**
3. External Risk Scout Agent **[LLM]**, modifiers bounded [-0.2, +0.3]
4. Risk Assessment **[deterministic]**, composite weights 0.35/0.35/0.30
5. Policy Agent **[deterministic]**, buffer posture
6. Warehouse Agent **[deterministic]**, hard constraint ≥0.75 → split
7. Decision Orchestrator Agent **[LLM]** → final JSON
- `use_deterministic_core` flag (default `True`) makes stages 4–6 rule-based; LLM used only for stages 1, 3, 7 ([orchestrator.py:408-456](agent-service/orchestrator.py#L408-L456)).
- README calls it a "5-Agent System"; code's `get_all_agents()` returns **7** agent objects ([agents.py:880-903](agent-service/agents.py#L880-L903)). See §16.

## 10. Core Logic

### agent-service (deterministic engine)
- `build_risk_profile(input_data)` — computes demand/supplier/warehouse/cash risk + effective lead time; prefers live `business_state`, falls back to payload ([risk_signals.py:363](agent-service/risk_signals.py#L363)).
- `calculate_demand_risk` / `calculate_supplier_risk` / `calculate_warehouse_stress` / `calculate_cash_risk` — categorical→numeric maps with live-data overrides ([risk_signals.py:109,188,265,334](agent-service/risk_signals.py#L109)).
- `compute_effective_lead_time_days` — `avg_lead_time × variability_multiplier` (LOW 1.1 / MEDIUM 1.3 / HIGH 1.6) ([risk_signals.py:37](agent-service/risk_signals.py#L37)).
- `calculate_composite_risk` — `0.35·demand + 0.35·supplier + 0.30·warehouse` ([decision_engine.py:56](agent-service/decision_engine.py#L56)); `classify_risk_level` thresholds LOW<0.4, MODERATE<0.7, HIGH≥0.7 ([decision_engine.py:98](agent-service/decision_engine.py#L98)).
- `determine_reorder_timing` (HIGH→EARLY, MODERATE→NORMAL, LOW→DELAYED) ([decision_engine.py:196](agent-service/decision_engine.py#L196)); `determine_order_strategy` (warehouse≥0.75→SPLIT_ORDERS, else cash≥0.7→FREQUENT_SMALL, else BULK) ([decision_engine.py:233](agent-service/decision_engine.py#L233)).

- `calculate_recommended_quantity` — EOQ-lite: `avg_daily_sales × lead_time × buffer_multiplier`, capped by available space; buffers MINIMAL 1.1 / MODERATE 1.25 / AGGRESSIVE 1.4 ([inventory_decision_agent.py:68](agent-service/inventory_decision_agent.py#L68)).
- `calculate_reorder_point` — ROP `= lead_time_demand × safety_multiplier`, capped at 90% capacity; status BELOW/NEAR/ABOVE; safety MINIMAL 1.0 / MODERATE 1.15 / AGGRESSIVE 1.30 ([inventory_decision_agent.py:177](agent-service/inventory_decision_agent.py#L177)).

- `_compute_risk_assessment`, `_compute_buffer_policy`, `_compute_warehouse_assessment`, `_clamp_external_modifiers`, `_generate_fallback_decision`, `_generate_default_context_summary` — deterministic helpers ([orchestrator.py:716-1038](agent-service/orchestrator.py#L716-L1038)).
